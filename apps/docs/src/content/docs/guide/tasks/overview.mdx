xtarterize applies conformance configuration through discrete, independently applicable tasks. Each task detects whether it's needed and only applies changes if necessary.

## Linting & Formatting

| Task ID | Description | Applicable When |
|---------|-------------|-----------------|
| `lint/biome` | Install Biome, write `biome.json` with framework-aware parser options | Always (TS/JS projects) |

<Aside type="tip">
  For stricter, opinionated linting, run <code>npx ultracite init</code> after <code>xtarterize init</code>. The <code>package.json</code> lint scripts will auto-detect Ultracite and use <code>ultracite:check</code> / <code>ultracite:fix</code> instead of Biome directly.
</Aside>

### Framework-Aware Biome Config

When Tailwind CSS is detected, the generated `biome.json` includes:

```json
{
  "css": {
    "parser": {
      "tailwindDirectives": true
    }
  }
}
```

This enables Biome to parse Tailwind v4 directives like `@theme` without errors.

### Conditional pnpm Setup

For projects using **pnpm**, CI workflows automatically include `pnpm/action-setup@v4` before `actions/setup-node`. This ensures reproducible pnpm versions across runs. npm and yarn projects are unaffected.

When `.vscode/extensions.json` already exists, xtarterize **adds** its recommended extensions to your existing list rather than replacing them. Your custom extensions are always preserved.

## Idempotency

<Aside type="tip">
  Every task is idempotent. Running `init` twice produces no changes on the second run.
</Aside>

Each task implements `check()`, `dryRun()`, and `apply()`:

- **`check()`** — Returns `new`, `patch`, `skip`, or `conflict` based on current state
- **`dryRun()`** — Returns exact diffs without writing anything
- **`apply()`** — Writes changes, backing up files first

## Task Gating

Tasks only run when their conditions are met:

- Vite plugin tasks only run when `bundler === 'vite'`
- Monorepo tasks only run when `monorepo === true`
- CI tasks only run when `hasGitHub === true`
- TypeScript tasks only run when `typescript === true`
